# 實作計畫

## 實作目標

建立一個前端版待辦事項 App，提供三個頁面：

- `全部`
- `進行中`
- `已完成`

並完成以下功能：

- 新增待辦事項
- 刪除待辦事項
- 標記待辦事項為完成

## 建議實作順序

### 1. 文件確認

- 以 `需求.md` 為唯一核心需求來源。
- 對照 `UI.md`、`流程.md`、`元件.md`，確認沒有超出既定範圍。

### 2. 專案初始化

- 建立 Web 前端專案。
- 預設套用深色模式樣式基礎。
- 建立頁面切換與共用版面結構。

### 3. 狀態模型建立

- 建立待辦事項資料結構：
  - `id`
  - `title`
  - `completed`
- 建立單一資料來源作為畫面狀態。
- 以篩選方式產生三個頁面的顯示內容。

### 4. 共用元件開發

- 頁面切換導覽
- 待辦事項輸入區
- 待辦事項列表
- 待辦事項卡片
- 空狀態區塊

### 5. 核心互動實作

- 新增待辦事項
- 切換完成狀態
- 刪除待辦事項
- 三頁資料同步更新

### 6. 響應式與視覺調整

- 手機版確認單欄操作順暢。
- 桌面版調整內容區寬度、留白與對齊。
- 確保預設為深色模式。

### 7. 驗收與測試

- 驗證三頁切換是否正確。
- 驗證新增、完成、刪除是否即時反映。
- 驗證同一份資料在不同頁面的顯示是否一致。
- 驗證空狀態與不同裝置尺寸下的畫面表現。

## 驗收條件

- 可以在任一頁面新增待辦事項。
- 未完成項目出現在 `全部` 與 `進行中`。
- 已完成項目出現在 `全部` 與 `已完成`。
- 刪除項目後，所有頁面都不再顯示該項目。
- 頁面預設使用深色模式。
- 手機與桌面版都能完成所有核心操作。
