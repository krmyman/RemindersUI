# AGENTS

## 文件一致性規則

- `需求.md` 為主。
- `UI.md` 必須與 `需求.md` 保持一致，不可自行擴增或改寫核心需求。

## 目前已確認的衝突

1. 預設主題
   - `需求.md`：預設使用深色模式
   - `UI.md`：預設為淺色模式
   - 結論：應以 `需求.md` 為準，UI 應改為預設深色模式

2. 頁面數量
   - `需求.md`：畫面上只有三個頁面
   - `UI.md`：總共有四個分頁
   - 結論：應以 `需求.md` 為準，UI 應改為三個頁面

3. 功能描述完整度
   - `需求.md`：包含新增待辦事項、刪除待辦事項、標記完成
   - `UI.md`：尚未描述上述功能如何在畫面中呈現
   - 結論：這是資訊缺漏，不是直接衝突，但後續 UI 文件應補齊

## 後續編輯要求

- 更新 `UI.md` 前，先對照 `需求.md`。
- 若文件衝突，先修正 `UI.md`，除非使用者明確表示 `需求.md` 已變更。
- 若新增頁面、主題設定或核心功能，需同步更新兩份文件。
