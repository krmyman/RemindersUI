# 流程

## 使用者主流程

### 1. 新增待辦事項

1. 使用者進入任一頁面。
2. 在主要操作區輸入待辦事項文字。
3. 點擊新增按鈕。
4. 系統建立一筆新的待辦事項資料，欄位至少包含 `id`、`title`、`completed`。
5. 新資料預設為未完成，因此：
   - 會出現在 `全部` 頁面。
   - 會出現在 `進行中` 頁面。
   - 不會出現在 `已完成` 頁面。

### 2. 標記待辦事項完成

1. 使用者在列表中找到某筆未完成待辦事項。
2. 操作該筆資料的完成控制。
3. 系統將該筆資料的 `completed` 狀態改為 `true`。
4. 畫面立即更新：
   - `全部` 頁面仍顯示該項目，但呈現為已完成狀態。
   - `進行中` 頁面不再顯示該項目。
   - `已完成` 頁面開始顯示該項目。

### 3. 刪除待辦事項

1. 使用者在任一頁面的列表中找到目標項目。
2. 點擊該筆資料的刪除按鈕。
3. 系統移除該筆待辦事項資料。
4. 畫面立即更新：
   - 當前頁面移除該項目。
   - 其他頁面也同步移除該項目。

### 4. 跨頁查看

1. 使用者透過頁面切換導覽在 `全部`、`進行中`、`已完成` 之間切換。
2. 系統使用同一份待辦資料，依頁面條件即時篩選顯示。
3. 使用者在任何頁面執行新增、完成、刪除後，切換到其他頁面時應看到一致結果。

## 畫面回饋規則

- 若輸入框為空白，新增操作不應建立資料。
- 若目前頁面沒有符合條件的項目，顯示空狀態文案。
- 所有操作結果都應立即反映在畫面上，不需要重新整理頁面。
