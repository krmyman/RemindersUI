# UI

- 這個 App 預設為深色模式。
- 畫面總共有三個頁面：`全部`、`進行中`、`已完成`。
- 三個頁面共用同一批待辦資料，只依照完成狀態篩選顯示，不各自維護獨立資料。
- UI 必須提供頁面切換操作，並全站使用一致的導覽方式。
- UI 必須提供新增待辦事項的操作入口。
- UI 必須提供刪除待辦事項的操作方式。
- UI 必須提供標記待辦事項完成的操作方式。

## 頁面定義

### 1. 全部

- 顯示所有待辦事項，不分完成狀態。
- 頁面上方提供標題與頁面切換導覽。
- 主要操作區提供待辦事項輸入框與新增按鈕。
- 列表中的每筆待辦事項都需顯示名稱、完成控制與刪除控制。

### 2. 進行中

- 只顯示尚未完成的待辦事項。
- 保留與其他頁面一致的頁面切換導覽。
- 新增待辦事項的操作入口仍可使用。
- 當項目被標記為完成後，該項目應立即自此頁消失。

### 3. 已完成

- 只顯示已完成的待辦事項。
- 保留與其他頁面一致的頁面切換導覽。
- 每筆待辦事項仍可被刪除。
- 若項目被改回未完成，該項目應立即自此頁消失。

## 操作呈現規則

- 新增待辦事項：
  - 使用文字輸入框搭配提交按鈕。
  - 不應接受空白內容。
- 標記完成：
  - 每筆待辦事項提供可直接操作的完成切換控制，例如勾選框或切換按鈕。
- 刪除待辦事項：
  - 每筆待辦事項提供刪除按鈕，點擊後立即移除資料與畫面內容。

## 狀態與回饋

- 當前頁面沒有符合條件的待辦事項時，需顯示空狀態文案。
- 深色模式為預設視覺，不提供主題切換需求。
- 版面需支援手機與桌面裝置：
  - 手機版以單欄清單為主。
  - 桌面版可放寬內容區，但不改變三頁資訊架構與操作流程。
